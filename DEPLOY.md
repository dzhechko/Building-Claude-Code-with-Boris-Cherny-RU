# Инструкция по публикации на GitHub Pages

## 🚀 Быстрый старт

Ваш сайт готов к публикации! Следуйте этим шагам:

### Шаг 1: Создайте репозиторий на GitHub

1. Откройте [GitHub](https://github.com/new)
2. Создайте новый **публичный** репозиторий
3. Назовите его как хотите, например: `boris-cherny-transcript`
4. **НЕ** инициализируйте репозиторий с README, .gitignore или лицензией
5. Нажмите "Create repository"

### Шаг 2: Настройте GitHub в этом окружении

**ВАЖНО:** Перед выполнением команд Git необходимо настроить авторизацию GitHub:

1. Перейдите на вкладку **#github** в интерфейсе sandbox
2. Выполните авторизацию GitHub App И OAuth
3. После успешной авторизации вернитесь сюда

### Шаг 3: Отправьте код в GitHub

После настройки GitHub выполните команды (замените `USERNAME` и `REPO` на свои):

\`\`\`bash
cd /home/user/webapp

# Добавьте remote репозиторий
git remote add origin https://github.com/USERNAME/REPO.git

# Отправьте код (используйте -f для первой отправки)
git push -f origin main
\`\`\`

### Шаг 4: Включите GitHub Pages

1. Откройте ваш репозиторий на GitHub
2. Перейдите в **Settings** → **Pages**
3. В разделе **Source** выберите:
   - Source: **GitHub Actions**
4. Сохраните изменения

### Шаг 5: Дождитесь деплоя

1. Перейдите на вкладку **Actions** в вашем репозитории
2. Вы увидите запущенный workflow "Deploy to GitHub Pages"
3. Дождитесь завершения (обычно 1-2 минуты)
4. После успешного деплоя ваш сайт будет доступен по адресу:
   
   **https://USERNAME.github.io/REPO/**

## 🎉 Готово!

Ваш сайт теперь опубликован и доступен в интернете!

## 🔄 Обновление сайта

Для обновления контента просто:

1. Внесите изменения в файлы
2. Закоммитьте изменения:
   \`\`\`bash
   cd /home/user/webapp
   git add .
   git commit -m "Update content"
   git push origin main
   \`\`\`
3. GitHub Actions автоматически пересоберёт и опубликует обновлённую версию

## 📝 Редактирование транскрипта

Весь текст транскрипта находится в файле:
- `/home/user/webapp/src/index.tsx`

Чтобы изменить текст:
1. Откройте файл в редакторе
2. Найдите нужный раздел (они помечены комментариями)
3. Внесите изменения
4. Пересоберите: `npm run build`
5. Закоммитьте и отправьте на GitHub

- `border-green-400` - зелёная граница
- и т.д.

Интерактивные функции находятся в:

### Стили
Дополнительные CSS стили можно добавить в секцию `<style>` в `src/index.tsx`

## 🐛 Решение проблем

### Workflow не запускается
- Убедитесь, что в Settings → Actions → General включены workflows
- Проверьте, что репозиторий **публичный**

### Сайт не открывается
- Дождитесь завершения первого деплоя (проверьте вкладку Actions)
- Проверьте, что в Settings → Pages выбран Source: "GitHub Actions"
- Подождите несколько минут после первого деплоя

### 404 ошибка при переходе по ссылкам
- Это нормально для GitHub Pages с динамическими маршрутами
- Убедитесь, что все ссылки начинаются с `#` (например, `#part1`)

## 📞 Нужна помощь?

Если возникли проблемы:
1. Проверьте логи в Actions на GitHub
2. Убедитесь, что все файлы закоммичены
3. Попробуйте локально: `npm run build && npm run preview`

## 🌐 Альтернативные способы публикации

### Cloudflare Pages
Если хотите использовать Cloudflare Pages вместо GitHub Pages:

1. Настройте Cloudflare API key через Deploy tab
2. Выполните:
   \`\`\`bash
   cd /home/user/webapp
   npm run deploy
   \`\`\`

### Vercel, Netlify
Сайт также совместим с Vercel и Netlify - просто подключите ваш GitHub репозиторий к этим платформам.
